# Agent Service UI 对话交互优化 Spec

## Why
当前对话 UI 更偏向日志流展示，缺少“先总览后分步”的过程可视化，用户难以快速理解当前进度与下一步动作。  
需要通过结构化步骤卡片与完成总结卡片，提升任务执行过程的可读性与可控感。

## What Changes
- 在对话开始阶段新增“整体执行计划”展示区域，明确端到端步骤概览。
- 将执行过程拆分为多个独立步骤卡片，并在每张卡片中显示当前步骤序号。
- 为每张步骤卡片增加可配置标题，用于标识该步骤目标。
- 为每张步骤卡片增加状态机：`开始`、`思考`、`计划`、`执行`、`完成`，并使用图标进行状态可视化。
- 在会话末尾新增“完成总结卡片”，突出交付结果与关键产出。
- 前后端事件字段对齐，确保状态流可被前端稳定消费与渲染。

## Impact
- Affected specs: Agent Service Web UI 对话渲染能力、SSE 事件语义能力
- Affected code: `agent_service/web/index.html`, `agent_service/web/app.js`, `agent_service/api/main.py`, `agent_service/agent/orchestrator.py`

## ADDED Requirements
### Requirement: 对话启动时展示整体执行计划
系统 SHALL 在新一轮对话开始时展示一张“整体执行计划”卡片，用于说明本轮任务预计步骤。

#### Scenario: 首轮消息触发计划展示
- **WHEN** 用户发起一轮新的任务请求
- **THEN** 前端在会话流中渲染一张整体执行计划卡片
- **AND** 卡片中至少包含步骤总数与步骤标题列表

### Requirement: 步骤卡片独立展示并包含序号与标题
系统 SHALL 将每个执行步骤渲染为独立卡片，并显示步骤序号与步骤标题。

#### Scenario: 执行进入某一步骤
- **WHEN** 后端发出某步骤的状态事件
- **THEN** 前端创建或更新对应步骤卡片
- **AND** 卡片显示“第 N 步”与该步骤标题

### Requirement: 步骤卡片状态机与图标可视化
系统 SHALL 为每张步骤卡片维护状态机，并对 `开始`、`思考`、`计划`、`执行`、`完成` 五种状态展示不同图标。

#### Scenario: 状态流式推进
- **WHEN** 同一步骤从一个状态变更为下一个状态
- **THEN** 前端更新该卡片的状态文本与图标
- **AND** 保留历史状态顺序，不回退到更早状态

### Requirement: 会话完成总结卡片
系统 SHALL 在一次任务执行完成后展示总结卡片，明确完成信号与关键结果。

#### Scenario: 任务结束
- **WHEN** 后端发送任务完成事件
- **THEN** 前端渲染明显的完成总结卡片
- **AND** 卡片包含完成标识、结果摘要与可访问产物信息（若有）

## MODIFIED Requirements
### Requirement: 流式消息渲染协议扩展
现有 SSE 协议 SHALL 扩展支持计划卡片、步骤卡片与完成总结卡片所需的结构化字段（如步骤 ID、步骤序号、步骤标题、状态枚举、状态图标键、总结内容），并与普通文本回复兼容。

## REMOVED Requirements
### Requirement: 无
**Reason**: 本次仅新增与扩展交互能力，不移除既有功能。  
**Migration**: 不涉及迁移。
