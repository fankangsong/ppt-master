我有一个基于 OpenClaw / Skill 的项目，目前 skill 是以函数/工具形式被调用的。

现在我要将这个 skill 改造为一个“可独立运行的 Agent 服务”，要求：

- Agent 内部可以调用自定义大模型（非默认模型）
- 对外提供 HTTP API
- 支持外部系统调用该 Agent

目标架构如下：

[Client] -> [HTTP API Server] -> [Agent] -> [LLM (可自定义)] -> [Skill逻辑]

说明：

- HTTP Server 负责接收请求
- Agent 负责：
  - 管理上下文（session/memory）
  - 调用 LLM
  - 调用原有 skill 逻辑
- LLM 支持配置：
  - base_url
  - api_key
  - model_name

请实现以下能力：

1. Agent封装
   - 将现有 skill 封装为 agent 的一个 tool 或内部能力
   - Agent 支持多轮对话（session_id）
2. 自定义模型接入
   - 支持 OpenAI-compatible API
   - 可配置参数：
     - endpoint
     - api_key
     - model
3. HTTP接口
   提供如下接口：

POST /chat
请求：
{
"session_id": "string",
"message": "string"
}

响应：
{
"reply": "string",
"usage": {},
"trace": {}
}

1. 可扩展性
   - 后续可以增加多个 skill/tool
   - Agent 支持 tool calling（函数调用）
2. 日志 & 调试
   - 打印 prompt
   - 打印 tool 调用过程

技术要求：

- 使用 Node.js（或 Python，选一个）
- 推荐框架：
  - Node.js: Express / Fastify
  - Python: FastAPI
- LLM调用使用：
  - OpenAI SDK 或兼容协议（axios/httpx）
- 代码结构清晰：
  - agent/
  - llm/
  - tools/
  - api/

请输出：

1. 完整项目结构
2. 每个文件的代码（可运行）
3. 如何启动服务
4. curl 调用示例
5. 如何配置自定义模型

增强要求：

- 支持 streaming（SSE 或 chunked response）
- 支持多 Agent（可扩展）

## 前端需求

需要提供一个 Web UI，用于调用 Agent HTTP 接口，实现类似 ChatGPT 的交互界面。

整体结构：

[Browser UI] -> [HTTP API Server] -> [Agent] -> [LLM]

UI 需要具备以下功能：

1. 聊天界面
   - 支持多轮对话
   - 展示用户消息 + Agent回复
   - 自动滚动
   - 支持 loading 状态

2. 输入能力
   - 输入框（支持 Enter 发送）
   - 支持长文本输入

3. 调用后端接口
   - 调用 POST /chat
   - 自动携带 session_id
   - 处理返回结果并渲染

4. 可视化调试信息（重要）
   - 可选展开：
     - prompt
     - tool 调用记录
     - token usage

5. 错误处理
   - 接口失败提示
   - 超时处理

6. 可选增强
   - 支持 streaming（逐字输出）
   - 支持 Markdown 渲染

前端技术要求：

- 使用 原生html
- UI 框架（任选其一）：
  - TailwindCSS（推荐）
  - 原生HTML

- 网络请求：
  - fetch 或 axios

- 代码结构：
  - index.html
  - components/
  - services/api.js
  - utils/
